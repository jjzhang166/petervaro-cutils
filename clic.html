CLIC | License Comment Generator


clic generates header-comments based on a file called INFO to all files implement a special comment at their beginnings. This file can contain a #format macro and special variables.

The comment syntax, which all files need to implement is the following (for actual examples check the INFO file's 2nd, 3rd and 4th examples): 


<token>     INFO      <char> ...  <char>      INFO     <token>
   ^    ^     ^    ^     ^    ^     ^    ^     ^    ^     ^
   |    |     |    |     |    |     |    |     |    |     |
   | optional | optional | anything | optional | optional |
   |  spaces  |  spaces  |          |  spaces  |  spaces  |
   |          |          |          |          |          |
opening      all     decorator  decorator     all      closing
symbol    uppercase  character  character  uppercase   symbol

Dependencies

Python 3


This module is written in Python 3, therefore it needs the interpreter to be installed on your system to run it. I suggest you to use a package-manager to keep your installations ordered, clean and upto-date.


Macintosh: one can use homebrew:


brew install python3

Windows: one can use chocolatey:


cinst python3

Linux: on Ubuntu and LinuxMint it is already installed:


sudo apt-get install python3

pyhashxx


It is an optional but highly recommended dependency. Under the hood clic is hashing the files, to detect if a file has been changed or not. Because pyhashxx is the fastest hashing algorithm, it will speed up the checking process. By default the fallback mechanism is SHA1 hashing which is way slower.


pyhashxx is available to install through pip:


pip3 install pyhashxx

Usage

Command Line Tool


clic is a command line tool that takes a path to a folder as a single argument, however it can also be used as a function imported from a Python module to customise the file extensions it will scan through searching for the special comment or to extend the language capabilities (adding more comment symbols).


Examples:

Using it as a command line utility on the current folder:


$ python3 clic.py .

Files

INFO file


This file doesn't have any extension and its name has to be INFO with all uppercase letters. This file contains the text which will be placed to any file in the provided folder (and in all sub folders) if the files define the a special comment which can be found in the following examples.

It can also implement the #format macro to specify the alignment and width of the generated comment.

It can also implement special variables which will be substitutated during the comment text generation. The syntax of the variables is to put them inside curly braces, for example: {VERSION} will import the content of the VERSION file and replace the placeholder with it.


Examples:

Example of an INFO file:


#format CENTER 64

My Wonderful Library
~~~~~~~~~~~~~~~~~~~~

File: {FILE} Size: {SIZE}
Update: {DATE}

Implemented by the one and only John Doe.
(C) 2014 John Doe. All rights reserved.

For each Python file which has the following comment at its beginning :


## INFO #
 # INFO ##

Will be replaced by this:


## INFO ########################################################
##                                                            ##
##                    My Wonderful Library                    ##
##                    ~~~~~~~~~~~~~~~~~~~~                    ##
##                                                            ##
##                File: file.py Size: 6.22 KB                 ##
##                     Update: 2014.07.05                     ##
##                                                            ##
##         Implemented by the one and only John Doe.          ##
##          (C) 2014 John Doe. All rights reserved.           ##
##                                                            ##
######################################################## INFO ##

For each C or H file which has the following comment at its beginning :


/* INFO *
 * INFO */

Will be replaced by this:


/* INFO ********************************************************
**                                                            **
**                    My Wonderful Library                    **
**                    ~~~~~~~~~~~~~~~~~~~~                    **
**                                                            **
**                 File: file.c Size: 2.47 KB                 **
**                     Update: 2014.07.05                     **
**                                                            **
**         Implemented by the one and only John Doe.          **
**          (C) 2014 John Doe. All rights reserved.           **
**                                                            **
******************************************************** INFO */

For each C or H file which has the following C99-style comment at its beginning :


// INFO //
// INFO //

Will be replaced by this:


// INFO ////////////////////////////////////////////////////////
//                                                            //
//                    My Wonderful Library                    //
//                    ~~~~~~~~~~~~~~~~~~~~                    //
//                                                            //
//                 File: file.h Size: 1.31 KB                 //
//                     Update: 2014.07.05                     //
//                                                            //
//         Implemented by the one and only John Doe.          //
//          (C) 2014 John Doe. All rights reserved.           //
//                                                            //
//////////////////////////////////////////////////////// INFO //

Format Macro

LEFT|CENTER|RIGHT


If one wants to specify the alignment of the generated header-comment, one has to provide the #format macro as the first token in INFO file with the desired aligntment, which are LEFT, CENTER or RIGHT. The default value is CENTER.


Examples:

Align the generated comments to the left:


#format LEFT

Align the generated comments to the right and limit width to 64 :


#format RIGHT 64

width


If one wants to specify the width of the generated header-comment, one has to provide the #format macro as the first token in INFO file with the desired width. The default value is 80.

Note: ccom won't wrap lines in INFO file, therefore if the provided width is smaller than the line widths those lines won't change.


Examples:

Specify width to 79 of the generated comments:


#format 79

Specify width to 100 and align CENTER the text of the generated comments :


#format CENTER 100

Variables

VERSION


Grab content of VERSION file if exists


LICENSE


Grab content of LICENSE file if exists


DATE


Get current date (YYYY.MM.DD)


SIZE


Get current file's size, convert and format it (B/KB/MB/GB)


FILE


Get current file's relative path from root + file name


FILE_NAME


Get current file's name with extension


FILE_BASE


Get current file's name without extension


Functions

clic.header ( str path , iterable of strs line , dict block , iterable of strs extensions )


clic can be used a module, which provides a module-level variable clic.EXTENSIONS and a public function called clic.header(). The latter can be used to customise the scanning and capturing settings.


Return: void


Examples:

Using clic.header to scan all HTML, Python and AppleScript files and capture single line comments start with -- and # and block comments start with <!-- ends with --> or start with (* ends with *) in the current folder and in all its subfolders :


from clic import header

header(path='.',
       line=('--', '#'),
       block={'<!--': '-->', '(*', '*)'},
       extensions=('.html', '.py', '.scpt'))
